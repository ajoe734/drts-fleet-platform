# UI Functional Contracts — 2026-05-24

## 1. Purpose

This document defines the 12 contracts engineering must add or amend in `packages/contracts/src/index.ts` or split contract modules.

The contracts are cross-surface primitives for Phase 1 UI behavior. They are not optional UI helper types. They express backend authority for refresh, health, actions, notifications, search, cross-app navigation, driver instructions, driver matching suppression, tenant readiness, and rollout state.

## 4. Endpoint impact map

| Contract | Minimum producer endpoints |
|---|---|
| `UiRefreshMetadata` | list/detail endpoints across all apps |
| `UiHealthEnvelope` | `/api/admin/health/ui`, `/api/driver/platform-presence/summary`, `/api/tenant/integration-readiness`, `/api/ops/dispatch/summary` |
| `ResourceActionDescriptor` | `*/actions` endpoints for tenant, ops, driver, admin resources |
| `EmptyStateEnvelope` | BFF/list responses where data is absent, gated, or not ready |
| `ActionReceipt` | all write endpoints |
| `UserNotificationRecord` | `/api/notifications` family |
| `CrossAppResourceLink` | detail endpoints for orders, tenants, partner entries, drivers, invoices, reports |
| `SearchResultRecord` | scoped search endpoints |
| `DriverOpsInstruction` | driver workspace, task detail, trip detail, ops driver control endpoints |
| `DriverMatchingSuppression` | driver profile, ops driver detail, platform presence, matching APIs |
| `TenantIntegrationReadinessSummary` | tenant console integration dashboard, platform-admin tenant detail |
| `TenantRolloutStateMachineRecord` | platform-admin tenant rollout detail |

## 5. Implementation acceptance

- All 12 contracts compile with `@drts/contracts`.
- API client exports typed accessors or response support where applicable.
- Each app uses at least three of these contracts in shipped surfaces.
- Unit tests cover serialization and minimal response examples for all 12 contracts.
- Integration tests cover `ActionReceipt`, `ResourceActionDescriptor`, and `UiRefreshMetadata` in at least one endpoint per app family.
