# Platform Admin Control Plane State Machines — 2026-05-24

## 1. Decision

Platform Admin state transitions must be backend-owned and represented through state-machine records and action descriptors. Platform Admin UI must not infer rollout, readiness, credential, pricing, or legal-hold actions locally.

## 2. Required contract: TenantRolloutStateMachineRecord

```ts
export type TenantRolloutStage = "sandbox" | "pilot" | "production" | "rollback_hold";
export type TenantRolloutGateStatus = "pending" | "ready" | "approved" | "blocked";

export interface TenantRolloutStateMachineRecord {
  tenantId: string;
  currentStage: TenantRolloutStage;
  gates: Array<{
    gateCode: string;
    label: string;
    status: TenantRolloutGateStatus;
    requiredEvidence: string[];
    blockers: string[];
    approvedByActorId?: string | null;
    approvedAt?: string | null;
  }>;
  nextActions: ResourceActionDescriptor[];
  rollback: {
    rollbackPrepared: boolean;
    cutoverOwner: string | null;
    rollbackOwner: string | null;
    lastRollbackAt?: string | null;
    rollbackActions: ResourceActionDescriptor[];
  };
  refresh: UiRefreshMetadata;
}
```

## 3. Tenant rollout state machine

```text
sandbox
  ├─ approve sandbox gates → pilot
  ├─ block → rollback_hold
pilot
  ├─ approve pilot gates + owners + rollbackPrepared → production
  ├─ block → rollback_hold
production
  ├─ incident / regression / operator decision → rollback_hold
rollback_hold
  ├─ resolve hold → previous safe stage
```

## 4. Partner credential state machine

```text
not_issued
issued
rotating
revoked
expired
compromised_hold
```

All credential transitions must return `ActionReceipt` and audit id.

## 5. Pricing publish state machine

```text
draft
review_required
scheduled
published
superseded
rollback_hold
```

Publishing requires:

- actor identity;
- version diff;
- effective time;
- reason where high-risk;
- audit receipt.

## 6. Vehicle / driver control-plane states

Vehicle dispatchability must respect:

- contract status;
- insurance validity;
- exclusivity review;
- debranding/offboarding task;
- maintenance state;
- regulatory holds.

Driver eligibility must respect:

- license/cert validity;
- device binding;
- shift state;
- platform binding;
- matching suppression;
- incident hold.

## 7. Adapter registry write split

- Platform Admin can configure adapter metadata, credentials, enabled state, and rollout.
- Ops Console can observe health and reconciliation states.
- Driver App only sees platform binding / driver-facing account state.
- Tenant Console only sees integration readiness where tenant-scoped.

## 9. Acceptance criteria

- `TenantRolloutStateMachineRecord` exported and returned by tenant detail APIs.
- Platform Admin uses backend-provided next actions for rollout and rollback.
- Partner credential, pricing publish, and rollback hold write actions return `ActionReceipt`.
- Platform Admin control-plane E2E validates tenant create → module enablement → quota → partner entry → credential → pricing publish → rollout → audit.
