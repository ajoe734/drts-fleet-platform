# Driver Platform Binding and Offline Contract — 2026-05-24

## 1. Decision

Driver platform binding and matching suppression are backend-owned. Driver App may display and request changes, but cannot authoritatively decide platform eligibility, matching suppression, or external-platform task authority.

## 5. Forwarded task authority

Forwarded tasks must keep:

- `sourcePlatform`;
- external order id;
- route locked flag;
- waypoints / route intent from source platform;
- accept pending state;
- lost-race state;
- platform cancellation state;
- sync failure state.

Driver App cannot start or complete a forwarded task unless backend action descriptor says that action is available.

## 6. Offline rules

Allowed offline:

- display stale cached tasks;
- display stale platform presence;
- draft proof uploads;
- queue telemetry.

Not allowed offline:

- accept external offer;
- reject external offer with authoritative relay;
- start forwarded task;
- complete forwarded task;
- clear lost-race/cancelled state;
- go online/offline on a platform.

## 7. Ops integration

Ops Console must use the same `DriverOpsInstruction` and `DriverMatchingSuppression` records for:

- suppressing matching during incident;
- forcing platform offline;
- acknowledging platform reauth required;
- handling manual fallback;
- reviewing driver eligibility.

## 8. Acceptance criteria

- Both contracts exported.
- Driver App renders instruction and suppression in workspace/platform/task detail.
- Ops Console can create or view suppression records through backend endpoints.
- E2E-006 or equivalent verifies mixed owned + forwarded task authority.
- No forwarded action can proceed without backend action descriptor.
