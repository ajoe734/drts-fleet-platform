# Search and Empty State Contract — 2026-05-24

## 1. Decision

Phase 1 does not implement global search across all apps. It implements scoped search and page-local search with typed result records.

Empty, not-ready, no-access, external-gated, sandbox-only, and configuration-missing states must be represented explicitly by backend or BFF metadata.

## 2. Required contract: SearchResultRecord

```ts
export interface SearchResultRecord {
  resultId: string;
  label: string;
  description?: string | null;
  resourceType: CrossAppResourceLink["resourceType"];
  resourceId: string;
  surface: CrossAppSurface;
  href: string;
  matchedFields: string[];
  tenantId?: string | null;
  score?: number | null;
  badge?: string | null;
  access: "allowed" | "forbidden" | "unknown";
  reasonCode?: string | null;
}
```

## 4. Search scope

| Surface | Search scope |
|---|---|
| Platform Admin | tenants, partner entries, users, fleet, pricing versions, audit events, adapters |
| Ops Console | orders, drivers, vehicles, complaints, incidents, call sessions, approvals, reconciliation issues |
| Tenant Console | bookings, passengers, addresses, cost centers, approval requests, invoices, reports, webhooks |
| Driver App | local task list, active trip, platform presence; no global search |
| Partner Booking | no general search; only booking lookup/tracking where allowed |

## 5. Empty-state examples

- No reports yet: `kind=empty`, primary action generate report if available.
- Tenant not production-ready: `kind=not_ready`, link to readiness summary.
- Adapter credential missing: `kind=configuration_missing`, link to partner entry credential action.
- Forwarder live proof unavailable: `kind=external_gated`, no misleading pass status.
- User lacks permission: `kind=no_permission`, no action except contact admin.
- Filter returns no result: `kind=filtered_empty`, offer clear filters.

## 6. API surface

```http
GET /api/platform/search?q=&scope=
GET /api/ops/search?q=&scope=
GET /api/tenant/search?q=&scope=
```

Search endpoints are optional where local page search is enough, but local page search still uses `SearchResultRecord` if results cross resource types.

## 7. Acceptance criteria

- `SearchResultRecord` and `EmptyStateEnvelope` exported.
- Each management app implements at least one empty state using `EmptyStateEnvelope`.
- Platform Admin and Ops Console implement at least one scoped search returning `SearchResultRecord`.
- Tenant Console booking/cost-center/report empty states use reason codes and backend action descriptors.
