# Tenant Console Migration Plan — 2026-05-24

## 1. Decision

`apps/tenant-console-web` is the repo-local canonical tenant admin console. External `tenant-commute-hub` remains live transition owner until a formal cutover record moves live traffic.

`apps/tenant-portal-web` remains sunset reference and must not be revived.

Partner Booking Mode moves to `apps/partner-booking-web`; `tenant-console-web/app/partner/*` is legacy compatibility / rollback reference only.

## 2. Surface roles

| Surface | Role |
|---|---|
| `apps/tenant-console-web` | repo-local canonical tenant admin console |
| `tenant-commute-hub` | current live transition owner until cutover |
| `apps/tenant-portal-web` | sunset reference only |
| `apps/partner-booking-web` | repo-local canonical white-label partner booking surface |
| `apps/tenant-console-web/app/partner/*` | legacy compatibility / rollback reference |

## 3. Canonical backend path

All tenant admin production data must come from `drts-fleet-platform` `/api/tenant/*` or typed BFF routes backed by that authority.

Tenant Console must not own:

- bookings;
- cost centers;
- approval rules;
- quotas;
- approval workflow;
- users / roles;
- API keys;
- webhooks;
- billing;
- reports;
- audit.

## 5. Cutover gates

Tenant Console live cutover requires:

1. tenant selected;
2. route/host ownership recorded;
3. cutover owner assigned;
4. rollback owner assigned;
5. rollback path verified;
6. login / booking / approval / billing smoke passed;
7. notifications and webhooks verified;
8. audit event written;
9. old surface retained for rollback retention period.

Partner Booking live cutover uses partner entry as migration unit, not whole tenant.

## 6. Required docs to update

- product spec amendment under `docs/01-product/`;
- partner booking cutover runbook if it exists;
- release gate matrix with tenant-console and partner booking non-claim language;
- tenant-console deployment docs.

## 7. Acceptance criteria

- `TenantIntegrationReadinessSummary` exported and available to Tenant Console and Platform Admin.
- Tenant Console has a readiness panel using this contract.
- External `tenant-commute-hub` transition state is documented; no silent retirement.
- Partner booking cutover remains explicitly separate from repo-local UI closeout.
